# Contributing Guidelines

## Branch Rules
- Do NOT push directly to `main`
- Create a new branch for each feature:
  - feature/feature-name
  - bugfix/bug-name

## Commit Messages
Use clear messages:
- feat: add user login
- fix: resolve auth bug
- refactor: clean API logic

## Pull Requests
- PR must describe changes clearly
- Link related Issue if exists
- At least 1 approval required

## Code Style
- Follow ESLint rules
- Run tests before pushing

## Environment Files
- Never commit `.env` files
